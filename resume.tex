\subsection*{Run Linux 6.19 on STM32H750 (Cortex-M7, no-MMU)}
\vspace{1pt}
Description: Run \textbf{Linux 6.19} on a microcontroller with only \textbf{1MB SRAM}\\
{\small\textbf{Links:} Github\hyperlink{l3}{[3]}\ \ Hackmd\hyperlink{l4}{[4]}}

\begin{itemize}
  \item Developed a \textbf{QEMU SoC model} with a core peripheral subset (UART, Memory Mapping, Timer), enabling \textbf{kernel debug via GDB}
  \item Developed a \textbf{minimal bootloader (only 12KB)} to load the Linux kernel
  \item Reduced Linux kernel .bss and .data usage to 100KB via custom kernel config
  \item Used \textbf{SPARSEMEM memory model} to utilize non-contiguous memory regions
  \item Used \textbf{romfs} to keep rootfs in flash
  \item \textbf{uClibc-ng} and toolchain integration for the portability of user program
  \item Used \textbf{bFLT} instead of ELF executables to run it without virtual memory support (no-MMU)
  \item User programs are \textbf{executed in place}, not using any RAM for text segment
  \item Linux kernel was successfully booted and run \textbf{toybox}, which is a lightweight busybox
\end{itemize}

\subsection*{Linux Kernel Instrumentation Framework (HITCON 2025)}
\vspace{1pt}
Description: Kernel module for dynamic instrumentation capable of intercepting kernel execution paths \textbf{without kallsyms or kprobes}. Presented at \emph{HITCON 2025}.\\
{\small\textbf{Links:} Github\hyperlink{l5}{[5]}\ \ Hackmd\hyperlink{l6}{[6]}}

\begin{itemize}
  \item Inspected and modified \textbf{procfs} file operations by parsing its internal tree structure
  \item Resolved syscall function addresses by dynamically analyzing the \textbf{syscall handler} in the Linux kernel (without relying on exported syscall table)
  \item Performed runtime kernel patching via \textbf{PTE flag manipulation}
  \item Achieved process concealment by manipulating the internal \textbf{PID hash} structure used by the scheduler
  \item Patched the \textbf{ELF header and entry point} to embed a minimal payload that invokes the kernel's module-loading syscall during systemd startup
\end{itemize}

\subsection*{Operating System running on STM32F103}
\vspace{1pt}
Description: A preemptive multitasking OS running on STM32F103 without the STM32 HAL.\\
{\small\textbf{Links:} Github\hyperlink{l9}{[9]}\ \ Hackmd\hyperlink{l10}{[10]}}

\begin{itemize}
  \item Reduced instruction fetch latency by executing the kernel entirely in SRAM
  \item Implemented \textbf{privilege separation} between kernel and user tasks with control register
  \item Built a \textbf{preemptive scheduler} supporting round-robin scheduling and \textbf{cooperative yielding}
  \item Implemented context switch using \textbf{setjmp/longjmp}
  \item Designed syscall mechanism scheduled as kernel tasks instead of running in handler mode
\end{itemize}
